# 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.x  | ✅ 积极维护 |

## 报告安全漏洞

如果您发现任何安全漏洞，**请不要在公开的 Issue 中报告**。

请通过以下方式私密报告：

1. 在 GitHub 仓库中创建一个 Issue，标题以 `[Security]` 开头
2. 或直接通过电子邮件联系项目维护者

报告时请提供以下信息：

- 漏洞类型和严重程度
- 复现步骤和环境
- 潜在影响范围
- 如果可行，提供修复建议

## 安全最佳实践

### 生产环境部署

- 使用防火墙限制对 etcd 端口的访问（默认 2379/2380）
- 不要在公网暴露 etcd 服务
- 定期备份数据目录
- 监控服务日志，注意异常访问

### 认证鉴权

当前版本暂未实现认证鉴权功能，建议在受信网络中运行。

## 安全更新

安全相关的修复将优先处理，并在 CHANGELOG.md 中标注为 `[Security]`。
